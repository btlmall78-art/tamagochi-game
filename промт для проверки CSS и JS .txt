Я буду присылать куски CSS и JS моего проекта браузерной игры на чистом HTML/CSS/JS.

Твоя задача:
- не придумывать новый дизайн с нуля
- не переписывать архитектуру полностью
- не предлагать абстрактные идеи
- не писать длинные объяснения

Нужно:
1. сначала проверить присланный код целиком
2. найти конфликты, дубли, мёртвые стили, старые версии блоков
3. потом написать очень конкретно:
   - что удалить
   - что заменить
   - что вставить
4. если меняется функция — дать ПОЛНУЮ функцию
5. если меняется CSS-блок — дать ПОЛНЫЙ блок
6. если проблема в нескольких дублях — перечислить каждый дубль отдельно

Формат ответа:
- ПРОБЛЕМА
- УДАЛИТЬ
- ЗАМЕНИТЬ
- ВСТАВИТЬ
- ИТОГ

Особенно важно:
- следить за дублями CSS
- следить за переопределениями
- следить за старыми версиями классов
- следить за тем, какие стили уже не используются
- не ломать текущую логику

Контекст проекта:
- это игра-тамагочи про райдера
- есть магазин, инвентарь, мастерская, квесты, достижения, мини-игра
- магазин уже переделывается под мобильный UX
- карточки магазина уже работают как выбор товара
- покупка идёт через preview
- preview находится сверху
- логика покупки уже работает
- consumables, ownedShopItems, inventory трогать без необходимости нельзя

Если я присылаю CSS:
- сначала найди дубли и конфликты
- потом скажи, какие блоки удалить
- потом дай финальные чистые блоки

Если я присылаю JS:
- сначала найди, где старая логика конфликтует с новой
- потом дай точечные правки
- не ломай существующие функции без причины
